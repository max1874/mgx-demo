#### **1. 产品概述**



- **产品名称**: AI 协同开发平台 (代号: MGX - DEMO)
-  **产品定位**: 一个顶级的 AI 智能体团队，允许任何用户通过自然语言，以聊天的方式协作，高效地创建网站、应用程序、数据报告、演示文稿等多样化的数字项目。
- **产品愿景**: 通过模拟一个高效、专业的软件开发团队，将复杂的软件开发生命周期（SDLC）自动化、智能化，让任何有好想法的人都能轻松、快速地将创意变为现实。



#### **2. 产品目标**



- **用户目标**:
  - **为非技术用户赋能**: 使不懂代码的用户（如创业者、市场人员、设计师）能独立构建功能完整的应用。
  - **为专业人员提效**: 帮助软件工程师、数据分析师等专业人士，从重复性工作中解放出来，专注于架构设计和核心业务创新。
  - **为团队协作减负**: 通过 AI 团队自动化执行，加速原型验证（MVP）和项目迭代，降低真人团队的沟通和管理成本。
- **业务目标**:
  -  **市场占有**: 成为 AI 原生应用开发领域的首选工具，在 AI 辅助开发平台中占据 “全流程覆盖” 与 “易用性强” 的理想区域。
  - **生态构建**: 建立一个活跃的 “应用世界 (App World)” 社区，用户可以分享、发现和 “改造 (Remix)” 项目，形成网络效应。
  - **商业模式**: 通过多层次的订阅计划（如专业版、Max 套餐）实现盈利，高级功能（如 Race Mode、移除 MGX 徽章）作为付费点。
  - **效率提升**: 利用多智能体协作机制，将传统软件开发周期缩短 70%。



#### **3. 目标用户与用户故事**



- **目标用户画像**:
  - **软件工程师**: 追求更快速、高效的工作流，用于快速搭建项目原型、前端页面或后端服务。
  - **非技术背景的创造者 (创业者 / 市场人员)**: 想要无代码 / 低代码创建个人博客、作品集、电商网站、个人链接页（Link Hub）等。
  - **产品 / 项目团队**: 需要快速制作可交互原型以进行市场验证和迭代。
  - **数据分析师**: 需要处理复杂数据集、生成数据洞察和可视化报告。
  - **学生与研究人员**: 用于学术研究、数据分析和制作答辩演示文稿（Slides）。
  - **技术爱好者与企业用户**: 希望观察和理解 AI 协作开发原理，或快速搭建内部工具以降低定制化开发成本。
- **用户故事**:
  - **作为创业者**: 我希望能够通过简单描述产品想法，快速生成可运行的原型，以便验证商业模式和获取早期用户反馈。
  - **作为产品经理**: 我希望系统能自动生成 PRD 文档和竞品分析报告，以便节省文档编写时间，专注于产品策略思考。
  - **作为开发者**: 我希望看到 AI 智能体如何协作完成复杂项目，以便学习最佳实践并应用到实际工作中。
  - **作为技术爱好者**: 我希望能够实时观察代码生成过程和智能体对话，以便理解 AI 辅助开发的工作原理。

#### **5. 核心功能 (Features & Functionalities)**





##### **5.1 核心交互：多智能体团队 (Multi-Agent Team)**



平台的核心是由多个职能明确的 AI 智能体组成的虚拟团队，他们会遵循标准化的操作流程（SOP）进行协作。

- **Mike (团队负责人)**: 项目总协调。负责协调用户需求，分配任务给最合适的成员，监督流程，并能从网络获取实时信息。
- **Emma (产品经理)**: 需求分析专家。负责将用户想法转化为结构化的产品需求文档 (PRD)，规划用户旅程和功能优先级。
- **Bob (系统架构师)**: 技术蓝图设计师。负责设计系统架构、技术栈选型、数据库结构和 API 接口，构建可靠、高效且可扩展的架构。
- **Alex (软件工程师)**: 全栈开发者。负责编写代码，实现从网站、游戏到仪表盘的各种应用。
- **David (数据分析师)**: 数据与 AI 专家。负责数据处理、模型训练、网页抓取和文档分析，将原始数据转化为洞察。
- **Iris (深度研究员)**: 专业研究员。负责执行深度研究任务，从多个可信来源收集信息，生成结构化、可溯源的专业报告。



##### **5.2 核心能力：多种工作模式 (Modes)**



为平衡速度、成本和功能复杂度，用户可以在不同模式间切换。

- **团队模式 (Team Mode)**:
  - **描述**: 默认模式，调用多个智能体协同工作。
  - **适用场景**: 复杂项目、通用需求、市场研究、数据分析等。
  - **特点**: 功能最全面，但积分消耗较高。
- **工程师模式 (Engineer Mode)**:
  - **描述**: 仅激活工程师 Alex 单一智能体。
  - **适用场景**: 简单的网站开发、快速原型制作。
  - **特点**: 速度快、效率高、积分消耗少。
- **深度研究模式 (Deep Research Mode)**:
  - **描述**: 调用研究员 Iris 进行深度调研，生成结构化、可溯源的专业报告。
  - **适用场景**: 学术论文、商业分析、市场报告等高精度内容。
  - **特点**: 成本高，仅在团队模式下可用，且仅支持特定的 LLM（如 gemini-2.5-pro）。
- **竞速模式 (Race Mode)** - *(高级功能 / Max 套餐)*:
  - **描述**: 同时调用多个不同的大语言模型（LLM）处理同一个指令，让用户选择最佳输出结果。
  - **适用场景**: 创意探索、文案生成、复杂推理等开放式任务。
  - **特点**: 极大提高获得优质结果的概率，但积分消耗也成倍增加。此功能仅向 Max 套餐用户开放。



##### **5.3 项目启动与创建**



- **自然语言输入**: 用户可在聊天框中直接输入提示词 (Prompt) 开始项目。
- **一键启动 (Click to Build)**:
  - **Slides**: 快速将想法或上传的文档（如论文）转化为精美的演示文稿。
  - **博客 (Blog)**: 一键生成带有基础模板的博客页面。
  - **个人链接页 (Link Hub)**: 快速生成用于社交媒体的个人信息聚合页。
- **改造 (Remix)**:
  - 用户可以从 “应用世界 (App World)” 或自己的历史记录中，免费克隆任何项目。
  - **价值**: 借鉴他人优秀项目；当一个项目版本过多导致上下文冗余时，通过 Remix 开启新对话以节省积分。



##### **5.4 开发与编辑过程**



- **实时可视化**:
  - **应用预览器 (App Viewer)**: 提供所生成应用的实时预览，支持 PC 和移动端视图切换。
  - **智能体协作区**: 实时展示各智能体的工作状态（工作中/等待中/已完成）。
  - **对话日志区**: 时间线形式展示智能体之间的对话和决策过程。
- **可视化编辑**: 允许用户在预览界面直接修改 UI 布局、组件样式和素材。
- **文件上传**: 支持上传文件/文件夹/图片作为智能体开发时的参考资料，以 `#文件名` 形式引用。
- **一键 Bug 修复**: 系统生成错误报告后，用户可点击 “Fix All” 按钮，由 AI 智能体自动尝试修复。



##### **5.5 集成与部署**



- **后端集成**: 与 **Supabase** 深度集成，自动生成无服务器后端架构。
- **版本控制**: 支持将代码直接提交到用户的 **GitHub** 或 **GitLab** 仓库。
- **发布 (Publish)**: 一键将应用部署到线上。支持绑定自定义域名。发布后，按钮变为 “更新 (Update)”，可选择部署最新或指定版本。专业版用户可移除 MGX 品牌徽章。



##### **5.6 分享与社区**



- **导出 (Export)**: 一键将整个项目（包含聊天记录和代码）下载为 `.zip` 文件。
- **权限控制**: 可将项目设置为公开、不公开或私密。
- **应用世界 (App World)**: 用户可将公开项目发布到社区，并自定义项目在社区中展示的封面、标题和描述。



#### **6. 技术规格**





##### **6.1 技术栈选型**



- **前端**: React + TypeScript, Shadcn-ui (UI 组件库), Tailwind CSS (样式), Monaco Editor (代码编辑器)。
- **后端 (可选)**: Node.js + Express, WebSocket (实时通信)。
- **AI 集成**: OpenAI API / Claude API (LLM), LangChain (智能体框架)。

#### **8. 非功能性需求**



- **性能**: 智能体响应和代码生成速度快，实时预览无明显卡顿。
- **可靠性**: 平台服务高度可用，确保用户项目数据安全不丢失。
- **安全性**: 生成的代码应避免常见的安全漏洞；用户数据和集成凭证需安全存储。
- **易用性**: 交互流程直观，引导清晰，非技术用户也能快速上手。
- **成本透明**: 积分消耗机制清晰可查。



#### **9. 成功指标 (Success Metrics)**



- **用户增长与活跃度**: 新用户注册数、日/月活跃用户 (DAU/MAU)。
- **核心功能使用率**: 各模式使用频率、项目创建和发布数量、Remix 功能使用次数。
- **社区健康度**: App World 中新发布的应用数量、应用被 Remix 的次数。
- **商业化指标**: 付费用户转化率、不同订阅套餐的收入占比。
- **用户满意度**: NPS (净推荐值)，从需求输入到代码预览的平均时间 < 2 分钟。
