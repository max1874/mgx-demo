模式概览

为了平衡速度、成本和功能，你可以选择工程师模式或团队模式。以下是区别：

 

工程师模式

单智能体：只会激活工程师智能体 @Alex。

快速且高效：专注于简单网站开发、快速原型和快速演示创建。

节省额度：相同的提示词，但额度使用更少。

 

团队模式

多智能体：调用多个智能体协作以应对复杂或通用需求。

更广场景：支持更多输出类型和案例，例如市场研究、数据分析等。

更高额度使用：由于功能更丰富、输出更多，额度使用更高。

 

查看每位团队成员的详细描述：你的智能体团队。

 

竞速模式（Race Mode）

选择最佳输出：多种 LLM 将同时处理你的请求。你可以比较多个输出并选择最佳一个。

 

深度研究模式（Deep Research Mode）

深度是关键：在深度研究模式中，你可以要求 AI 智能体从多个资源收集信息并为你撰写一份深入报告。

 

如何使用

切换模式

在首页输入框左下角点击模式切换按钮。

在弹出菜单中选择工程师或团队。

新聊天将记住你的选择。

 

随时切换

你可以在聊天前或聊天过程中随时切换模式，以满足变化的需求。

 

查看聊天模式

在 App World 中，你可以在项目详情页的标签栏查看每个项目所使用的模式（工程师或团队）。

Race Mode（竞速模式）
让多个 LLM 同台竞技，然后选择最佳输出。

超过一个月前更新
 

什么是 Race Mode
不同的 LLM 在知识、推理、文风上各有优势。

Race Mode 允许你同时向多个模型发送同一条指令并比较它们的输出，

通过并行生成答案来提高成功率，快速找出最理想的结果。

 

你可以在 Race Mode 中选择多种 LLM。

📃** 提示：** 在 Choosing LLMs 页面可以进一步了解它们的差异。

 

 

使用方法
步骤 1

在首页聊天框底部选择 Race Mode。

 

步骤 2

开始聊天前，可以设置需要参与构建的其他 LLM。

默认模式是 Auto，点击 Auto 可自定义 LLM。

 

步骤 3

点击齿轮图标。

齿轮图标会打开 AI 模型选择弹窗。

 

步骤 4

等待 5–10 分钟，选择最优输出。

 

步骤 5

选定最佳结果后，其他结果会被自动丢弃以节省存储空间。

 

可选步骤

点击界面顶部的 Stop 按钮可随时停止 Race Mode。

停止后你可以：

直接选择自己喜欢的结果；

或让智能体自动选择一个结果。

 

注意事项
如果项目已启用 Supabase 或 Stripe 集成，则无法使用 Race Mode。

Race Mode 比标准模式消耗更多 积分。

请提前检查积分余额，以免任务中断。

 

Race Mode 的优势
在复杂项目中，Race Mode 可以帮助你：

更快完成工作：一次运行多个 LLM，快速测试想法。

获得更优结果：并行尝试多种方案，提高找到最佳输出的概率。

挑选最优选项：比较不同模型的优劣，选择你最满意的结果。

 

最佳使用阶段
探索创意：让多个 LLM 同时生成更多点子，加速头脑风暴。

原型迭代：多个模型并行修复错误，实现快速迭代。

测试阶段：可同时生成多份文案、提示词或 UI 设计，方便挑选最佳方案。

评估模型：并行运行主流 LLM，对比其表现，为项目挑选最合适的模型。

降低幻觉风险：多模型交叉验证，减少错误或虚构信息。

 

常见问答（FAQ）
问：何时使用 Race Mode？

可用两种方式：

同一 LLM、多版本：并行运行同一个模型，获取更多样化结果，更快找到最佳方案。

多种 LLM 同时运行：在复杂任务中让不同模型协同，发挥各自优势。

问：我是新手，适合用 Race Mode 吗？

是否适用取决于项目复杂度和目标，而不是经验。

简单项目：普通模式足够。

需要高准确度、稳定性或快速比较不同 LLM 时：Race Mode 更有效。

可查看我们准备的入门指南 Quick Start。

问：选择 Race Mode 后还能像平常一样使用 MGX 吗？

可以。选择最佳结果后，你即可像平时一样继续使用 MGX。

⚠️ 但若项目与 Supabase 或 Stripe 集成，将无法使用 Race Mode。

 

问：没有 Max 套餐能体验 Race Mode 吗？

抱歉，Race Mode 仅向 Max 套餐用户开放，因为同时运行多个 LLM 成本较高。

 

问：Race Mode 对项目类型或文件大小有限制吗？

没有，所有类型和大小的项目都可使用。

但它消耗更多积分，若积分不足，项目生成可能失败。

 

问：Race Mode 真的比普通模式好很多吗？

开放式任务（如文案撰写、创意生成、复杂推理）：多模型输出差异大，可得到更佳结果。

封闭式任务（如事实验证、数据检索）：不同模型往往得出相同答案，优势不明显。

什么是 深度研究 Deep Research（DR）
Deep Research（简称 DR）是一项面向专业报告的功能，涵盖学术论文、商业分析、市场报告等场景。

MGX 的深度研究员 Iris 会接管你的复杂任务，通过检索信息、筛选可靠来源并生成可供审阅的报告。

 

Iris 的报告结构化、可溯源且可靠：

每一条观点都有可信来源支撑，避免 AI 幻觉。

只需一键，即可将报告转化为网站、PPT、PDF 或 Docs。

DR 不是普通搜索引擎，而是专业研究员和精准写手。

当你需要可靠信息和可交付成果时，就是 @Iris 使用 DR 的最佳时机。

 

⚠️ 注意

DR 成本较高，仅在 Team Mode（团队模式）下可用。

仅支持 gemini-2.5-flash 与 gemini-2.5-pro 模型。

更适合复杂项目，而非日常提问。

适用场景
高精度需求：学术研究、商业分析、市场调研或任何需要专业报告的任务。

可交付成果：需要生成文档、PPT 或网站。

团队协作：DR 默认仅可在 团队模式 下启用。

复杂策略：相比普通聊天，DR 提供更结构化、可靠的调研框架。

 

重要提示
DR 仅在团队模式下可用。

开启 DR 后，其他模型需关闭 DR 才可使用。

DR 与 Race Mode 不兼容。

 

常见问题（FAQ）

Q：可以直接使用输出的报告吗？

A：可以。输出结果可一键转化为网站或 PPT，并支持下载。

 

Q：在 DR 中可以 @ 其他智能体吗？

A：可以。Iris 会与其他智能体协作，以获得更优的生成结果。

 

Q：DR 适合用来获得快速答案吗？

A：不建议。对于简单搜索，DR 可能显得慢、成本高且流程复杂。它更适合高精度、可交付的专业场景。

什么是 Remix
超过 3 周前更新
您可以使用 Remix 在自己的对话中获得一个一模一样的项目，并在新项目中进行编辑。它相当于在克隆的基础上继续搭建、修改。

Remix 任何项目都是 0 成本，不会扣除积分（credits）。

 

怎么使用 Remix
您可以从 App World, Chat, 历史记录中对当前项目进行 Remix。

App World：查看公开项目时，页面右上角有 Remix 按钮。Remix 时可自定义项目名称。

Chat：在版本高亮框的右上角点击 Remix 按钮。

历史记录：在历史记录中选择特定版本，点击 Remix。

使用场景：
借鉴他人项目

在 App World 中看到优质项目时，您可以通过 Remix 获得副本并继续编辑。

Step 1: 在页面右上角点击 Remix。

Step 2: 在弹窗中确认，并可修改项目名称。

Step 3: Remix 成功后，会在 Chat 中打开新项目，同时仍可在原始版本的高亮框中查阅初始构建过程。

 

节约成本

当一个 Chat 内版本过多（如 10 个以上），由于 LLM 需要重复读取历史对话，会导致 credits 消耗加快。

建议此时进行 Remix，在新的对话中继续工作。

Step 1: 点击显示版本数的高亮框中的 Remix 按钮。

Remix 有什么好处？
节省消耗：在多版本对话中，Remix 可减少上下文冗余，降低积分花费并提升效果。

借鉴模范：在 App World 中 Remix，他人的优质公开项目即可免费获得，并可自定义项目名称。
